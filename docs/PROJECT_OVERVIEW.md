# 项目总览

## 新目标
构建一个由 LLM 与人类共同驱动的可审计策略工厂，而不是仅靠固定策略模板和参数调节运行的交易程序。

补充背景见：
- `PROJECT_CONTEXT.md`

## 核心原则
- LLM 负责提出策略和解释市场
- `RiskManagerAgent` 拥有 B 级自治权，决定是否进入或留在模拟盘
- `ExecutionAgent` 必须是确定性执行层
- 审计链路必须可以回答“当时为什么允许这件事发生”

## 回测 / 模拟盘 / 实盘定义
- 回测：用历史数据离线验证策略，不产生真实订单
- 模拟盘：用在线或准实时数据运行策略，生成模拟订单与净值，但不动真实资金
- 实盘：通过券商接口使用真实资金下单

## B 级自治边界
- 自动创建研究任务
- 自动运行回测、walk-forward、研究辩论
- 满足门禁后自动进入模拟盘
- 不自动进入实盘

## 实盘准入方向
- `v1` 目标：把研究、治理、模拟盘和审计闭环做实
- `v2` 目标：证明系统长期稳定，形成明确实盘准入证据
- `v3` 目标：在小资金、强风控、可回滚前提下进入受控实盘

当前运行基线：
- 一台长期在线的 `Mac mini`
- `launchd` 守护本地服务
- dashboard 作为评审看板，而不是实盘控制台

当前 `v2` 的推进顺序已经固定：
1. 先固化 `LLM provider` 迁移治理原则，保证未来换模型时历史证据不丢、对照可做
2. 再讨论桌面应用封装，把当前 Mac mini 本地控制面产品化

当前默认原则：
- 模拟盘成功不等于可进实盘
- 进入实盘必须满足独立的准入标准
- 实盘启动必须保留人工确认，而不是自动升级

## 北极星指标
- 所有策略都有证据包与审计记录
- 模拟盘前必须有结构化放行结果
- 风险红线触发后必须可自动暂停或回滚
- 宏观日摘要已进入上下文和软评分
