# OpenHamster v0.2.0

## Summary
`v0.2.0` 是 OpenHamster 公开发布前的统一重命名与仓库整理版本。

这一版的重点不是扩更多市场或执行链路，而是把当前主线的品牌、开源入口、文档和运行命名全部收口到同一个对外形态：
- HK-only
- dynamic HK universe selection
- MiniMax live strategy research
- deterministic governance
- local paper ledger
- operator dashboard
- English-first public repository and Pages

## Highlights

### Full public rename
- Python package 已统一为 `openhamster`
- CLI 已统一为 `openhamster-api`
- launchd label 已统一为 `com.openhamster.api`
- 前端标题、配置默认值、Prompt 契约文案、README 和 Pages 已统一为 `OpenHamster`

### Public repository cleanup
- 仓库公开入口已统一为：
  - GitHub: `https://github.com/nullwuwu/openhamster`
  - Pages: `https://nullwuwu.github.io/openhamster/`
- 公开文档已清除旧项目名、旧仓库路径和本机绝对路径
- 旧的发布叙事已从过时的 agent-trading / broker / MCP 历史切回当前产品主线

### Current product baseline
- 保留 HK-only 研究与模拟盘主线
- 保留 `dynamic_hk` 自动选股
- 保留 `LLM Gateway` + MiniMax / mock provider 结构
- 保留确定性治理、审计链与本地 paper ledger
- 继续明确不做 broker execution 和 automatic live trading

## Included Operator Views
- `/command`
- `/candidates`
- `/research`
- `/paper`
- `/audit`

## Validation Baseline
- `pytest tests -q` -> `122 passed, 8 skipped`
- `npm run build --prefix apps/web` -> passed

## Known Limits
- SQLite is still the default delivery database
- paper execution is simulated, not broker-connected
- long-horizon operating evidence still needs more runtime history
- macro context intentionally excludes news and announcements

## Upgrade Notes
- 旧命名 `gobyshrimp` 已不再保留兼容别名
- 如果本地脚本、launchd 或外部自动化仍引用旧 CLI 或旧 plist 名称，需要同步切换
- 当前发布重点仍然是研究工厂与模拟盘治理，不是自动实盘
