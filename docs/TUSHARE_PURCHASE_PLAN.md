# Tushare 购买清单

## 目标
给 GobyShrimp 当前 `HK-only（仅港股）` 主线确定一个最小、够用、分阶段的 `Tushare（数据权限）` 购买方案。

当前系统重点是：
- `backtest（回测）`
- `paper trading（模拟盘）`
- `live readiness（实盘就绪度）` 证据积累

当前不是分钟级高频，也不是财报驱动策略。

## 第一阶段：现在就值得买
### 1. 港股历史日线
- 页面价格：`1000元/年`
- 对应接口：`hk_daily（港股历史日线）`
- 用途：
  - `backtest（回测）`
  - `paper trading（模拟盘）` 历史估值
  - `universe ranking（选股排序）`
- 结论：
  - 这是当前最值得先买的权限。

### 2. 2000 积分档
- 页面价格：`200元/年`
- 主要用于覆盖：
  - `hk_tradecal（港股交易日历）`
  - `hk_basic（港股基础信息）`
- 用途：
  - 判断今天是否交易日
  - 判断节假日 / 周末 / 非交易时段是否应跳过成交
  - 获取港股列表、每手股数、币种、可交易性信息
- 结论：
  - 如果你希望交易日判断和标的基础信息也统一走 `Tushare（数据源）`，这档积分值得一起配上。

## 第二阶段：模拟盘跑通后再买
### 3. 港股实时日线
- 页面价格：`1000元/月`
- 对应接口：`港股实时日线`
- 用途：
  - 更及时的 `latest_price（最新价格）`
  - 更及时的 `NAV（净值）` 更新
  - 更接近交易日内真实运行的估值体验
- 结论：
  - 当模拟盘主线跑稳后，这是下一步最值得补的权限。

注意：
- 如果当前主运行标的是 ETF（交易所买卖基金），例如 `2800.HK（盈富基金）`
- 普通 `港股实时日线` 不一定覆盖它
- 这时更可能需要单独的 `ETF实时日线` 权限
- 在补齐该权限前，系统应继续允许：
  - `实时接口失败 -> 回退到历史日线最近收盘价`

## 第三阶段：需要更细粒度盘中研究时再买
### 4. 港股历史分钟
- 页面价格：`2000元/年`
- 对应接口：`港股历史分钟`
- 用途：
  - 更细粒度的盘中回测
  - 更细的 `paper trading（模拟盘）` 盘中行为验证
- 结论：
  - 当前不是刚需。
  - 只有当策略真的要进入分钟级研究或盘中验证时再买。

## 当前不急
### 5. 港股财报
- 页面价格：`500元/年`
- 当前主线不是财报驱动策略
- 暂不优先

## 推荐购买顺序
1. `港股历史日线`
2. `2000 积分档`
3. `港股实时日线`
4. `港股历史分钟`
5. `港股财报`

## 对应到代码接入顺序
1. `hk_daily（港股历史日线）`
2. `hk_tradecal（港股交易日历）`
3. `hk_basic（港股基础信息）`
4. `港股实时日线`
5. `港股历史分钟`

## 一句话结论
当前最小可用组合是：
- `港股历史日线`
- `2000 积分档`

模拟盘跑通后，下一步最值得补的是：
- `港股实时日线`

如果主运行标的是 ETF（交易所买卖基金），则需要进一步确认：
- 是否改买 `ETF实时日线`
- 或继续接受 `历史日线回退` 作为当前阶段的估值策略
